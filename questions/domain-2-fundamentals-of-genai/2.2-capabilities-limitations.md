## Q115

### 🇺🇸 Question (English)
An LLM was trained on data up to a certain date. When asked about a news event that happened AFTER that date, it may provide outdated or fabricated information. Which mitigation is MOST DIRECT?

### 🇰🇷 문제 (한글)
LLM은 특정 시점까지의 데이터로 학습되었습니다. 그 이후에 일어난 뉴스 사건에 대해 질문받으면, 오래되거나 지어낸 정보를 줄 수 있습니다. 가장 **직접적인** 완화책은 무엇입니까?

### Options / 보기
- **A.** Increase model temperature to encourage creativity.
  - 한글: 창의성을 유도하도록 모델 temperature를 높임.
- **B.** Delete the model and retrain from scratch each day.
  - 한글: 매일 모델을 삭제하고 처음부터 재학습.
- **C.** Use Retrieval Augmented Generation (RAG) to inject fresh, up-to-date context from a live knowledge base into the prompt at inference time.
  - 한글: 추론 시점에 최신 지식 베이스에서 신선한 문맥을 프롬프트에 주입하도록 RAG(검색 증강 생성)를 사용.
- **D.** Disable Guardrails so the model can speak freely.
  - 한글: 모델이 자유롭게 발언하도록 가드레일 비활성화.

### ✅ Correct Answer / 정답
**C**

### 💡 Explanation / 해설

**English**:
RAG directly addresses knowledge-cutoff by retrieving current information at inference. A worsens accuracy. B is prohibitively expensive. D creates safety issues.

**한글**:
RAG는 추론 시점에 최신 정보를 검색해 지식 컷오프 문제를 직접 해결합니다. A는 정확도 악화, B는 비용이 감당 불가, D는 안전 이슈를 유발합니다.

## Q116

### 🇺🇸 Question (English)
A team runs the same LLM prompt twice and receives two slightly different responses. Which factor is MOST likely responsible for this non-determinism?

### 🇰🇷 문제 (한글)
한 팀이 같은 LLM 프롬프트를 두 번 실행했는데 약간 다른 응답 두 개를 받았습니다. 이러한 비결정성에 가장 큰 원인이 될 가능성이 높은 요소는 무엇입니까?

### Options / 보기
- **A.** The AWS region changed automatically.
  - 한글: AWS 리전이 자동으로 변경됨.
- **B.** IAM policies expire per request.
  - 한글: IAM 정책이 요청마다 만료됨.
- **C.** S3 was throttled.
  - 한글: S3가 쓰로틀링됨.
- **D.** Sampling parameters (temperature > 0, Top-P, Top-K) introduce stochasticity into token selection, so repeated calls can produce different outputs.
  - 한글: 샘플링 파라미터(temperature > 0, Top-P, Top-K)가 토큰 선택에 확률성을 도입하기 때문에, 반복 호출이 서로 다른 출력을 낼 수 있음.

### ✅ Correct Answer / 정답
**D**

### 💡 Explanation / 해설

**English**:
Non-determinism arises from sampling. To make outputs (nearly) reproducible: temperature=0 and greedy decoding. A, B, C are unrelated AWS operational issues.

**한글**:
비결정성은 샘플링에서 비롯됩니다. (거의) 재현하려면 temperature=0과 그리디 디코딩을 씁니다. A·B·C는 무관한 AWS 운영 이슈입니다.
