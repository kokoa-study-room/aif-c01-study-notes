AI 컴플라이언스·거버넌스에 학습 데이터의 출처와 처리 과정을 아는 **데이터 프로버넌스**가 중요한 이유는 무엇입니까?

- **B.** Because CloudFront requires it.
  - 한글: CloudFront가 요구하기 때문.
- **C.** Because provenance enables verifying that data was obtained legally, subjects consented as required, licenses permit this use, sensitive classes were handled correctly, and (when regulators or subjects ask) the organization can trace which model versions used which data — foundational for GDPR, HIPAA, EU AI Act, and vendor risk management.
  - 한글: 프로버넌스가 데이터가 합법적으로 확보되었는지, 정보 주체가 필요한 동의를 했는지, 라이선스가 이 사용을 허용하는지, 민감 클래스가 올바르게 처리되었는지, 그리고 (규제 기관이나 정보 주체가 물을 때) 어느 모델 버전이 어떤 데이터를 사용했는지 조직이 추적할 수 있게 함을 검증할 수 있게 함 — GDPR·HIPAA·EU AI Act·벤더 리스크 관리의 기초.
- **D.** Because IAM refuses to work otherwise.
  - 한글: 그렇지 않으면 IAM이 동작을 거부하기 때문.

### ✅ Correct Answer / 정답
**C**

### 💡 Explanation / 해설

**English**:
Provenance underpins consent, legality, licensing, sensitivity handling, and traceability — key for GDPR/HIPAA/EU AI Act. A, B, D are false.

**한글**:
프로버넌스는 동의·합법성·라이선스·민감성 처리·추적성의 기초이며, GDPR·HIPAA·EU AI Act에 핵심적입니다. A·B·D는 사실 아닙니다.
