## Q321

### 🇺🇸 Question (English)
Which BEST distinguishes "interpretability" from "explainability" in ML?

### 🇰🇷 문제 (한글)
ML에서 "해석 가능성(interpretability)"과 "설명 가능성(explainability)"을 가장 잘 구분하는 설명은 무엇입니까?

### Options / 보기
- **A.** Interpretability describes how INHERENTLY understandable the model is (e.g., a small decision tree). Explainability describes techniques used to explain a model's outputs even if the model itself is complex/opaque (e.g., SHAP for a neural network). Both aim to make model behavior humanly understandable.
  - 한글: 해석 가능성은 모델이 얼마나 **본질적으로** 이해 가능한지(예: 작은 의사결정 트리)를 말함. 설명 가능성은 모델 자체가 복잡·불투명하더라도 출력을 설명하기 위한 기법(예: 신경망에 대한 SHAP)을 말함. 둘 다 모델 동작을 사람이 이해할 수 있도록 하는 것이 목표.
- **B.** They are the same thing.
  - 한글: 둘은 같은 것.
- **C.** They are unrelated to ML.
  - 한글: 둘 다 ML과 무관.
- **D.** Interpretability means faster; explainability means slower.
  - 한글: 해석 가능성은 더 빠름, 설명 가능성은 더 느림.

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
Interpretability = inherently understandable; explainability = techniques to explain opaque models. B, C, D are wrong.

**한글**:
해석 가능성 = 본질적 이해 가능성; 설명 가능성 = 불투명 모델을 설명하는 기법. B·C·D는 사실 아닙니다.

## Q329

### 🇺🇸 Question (English)
Which BEST distinguishes GLOBAL explanations from LOCAL explanations?

### 🇰🇷 문제 (한글)
