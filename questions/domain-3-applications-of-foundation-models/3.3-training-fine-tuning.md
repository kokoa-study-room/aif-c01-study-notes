## Q233

### 🇺🇸 Question (English)
Which statement BEST distinguishes "pre-training" from "fine-tuning"?

### 🇰🇷 문제 (한글)
"사전 학습(pre-training)"과 "파인튜닝(fine-tuning)"을 가장 잘 구분하는 진술은 무엇입니까?

### Options / 보기
- **A.** Pre-training builds a general foundation model on a huge, diverse corpus (very expensive). Fine-tuning then adapts that model to a specific domain or task with a much smaller dataset.
  - 한글: 사전 학습은 방대하고 다양한 코퍼스로 범용 파운데이션 모델을 구축(매우 비쌈). 파인튜닝은 훨씬 작은 데이터셋으로 그 모델을 특정 도메인이나 작업에 적응시킴.
- **B.** They are the same thing.
  - 한글: 둘은 같은 것입니다.
- **C.** Fine-tuning is more expensive than pre-training.
  - 한글: 파인튜닝이 사전 학습보다 비쌉니다.
- **D.** Pre-training only works on images.
  - 한글: 사전 학습은 이미지에만 동작합니다.

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
Pre-training is the expensive one-time process; fine-tuning is cheaper task/domain adaptation. B, C, D are false.

**한글**:
사전 학습은 비싼 1회성 과정이며, 파인튜닝은 저렴한 작업/도메인 적응입니다. B·C·D는 사실 아닙니다.

## Q237

### 🇺🇸 Question (English)
A team fine-tunes an FM heavily on one narrow classification task and notices the model has become worse at other tasks it used to handle well. What is this phenomenon called?

### 🇰🇷 문제 (한글)
한 팀이 FM을 하나의 좁은 분류 작업으로 강하게 파인튜닝한 뒤, 이전에 잘하던 다른 작업들에서 성능이 저하되었음을 관찰합니다. 이 현상은 무엇이라 부릅니까?

### Options / 보기
- **A.** Catastrophic forgetting — over-specialization to a narrow task erodes the model's general capabilities learned in pre-training.
  - 한글: 파국적 망각(Catastrophic forgetting) — 좁은 작업에 과도하게 특화되면서, 사전 학습에서 습득한 일반 능력이 침식됨.
- **B.** Overfitting to the AWS region.
  - 한글: AWS 리전에 대한 오버피팅.
- **C.** Underfitting.
  - 한글: 언더피팅.
- **D.** Data drift.
  - 한글: 데이터 드리프트.

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
Catastrophic forgetting is the loss of general capabilities during aggressive narrow fine-tuning. B, C, D are unrelated.

**한글**:
파국적 망각은 좁은 파인튜닝 중 일반 능력을 상실하는 현상입니다. B·C·D는 무관합니다.

  - 한글: 모든 가중치를 업데이트하는 대신, 베이스 모델은 고정한 채 소수의 새 파라미터(어댑터) 또는 저계수 업데이트만 학습 — 훨씬 저렴·빠르고 태스크별 저장이 쉬움.

  - 한글: 암호화된 데이터로만 학습.
- **D.** Disabling gradient descent.
  - 한글: 경사 하강 비활성화.

**English**:
PEFT trains a small adapter while freezing the base — cheaper storage and compute. A, C, D are false.

**한글**:
PEFT는 베이스를 고정하고 작은 어댑터만 학습해 저장·컴퓨트가 저렴합니다. A·C·D는 사실 아닙니다.

### 🇺🇸 Question (English)
After a Bedrock fine-tuning job succeeds, which is required to serve INFERENCE from the resulting custom model at production scale?

### 🇰🇷 문제 (한글)
Bedrock 파인튜닝 작업이 성공한 뒤, 결과 커스텀 모델의 **추론**을 프로덕션 규모로 서빙하려면 필요한 것은 무엇입니까?

### Options / 보기
- **A.** Delete base models.
  - 한글: 베이스 모델 삭제.
- **B.** Turn off IAM.
  - 한글: IAM 비활성화.
- **C.** Purchase Provisioned Throughput for the custom model, sized to expected traffic.
  - 한글: 예상 트래픽에 맞춘 크기로 커스텀 모델의 Provisioned Throughput 구매.

  - 한글: 다른 클라우드로 이관.

**C**

Custom models on Bedrock require Provisioned Throughput to serve. A, B, D are wrong.

**한글**:
Bedrock의 커스텀 모델은 서빙에 Provisioned Throughput이 필요합니다. A·B·D는 사실 아닙니다.

## Q260

### 🇺🇸 Question (English)
Which BEST distinguishes TRAINING cost from INFERENCE cost for foundation models?

### 🇰🇷 문제 (한글)
파운데이션 모델의 **학습** 비용과 **추론** 비용을 가장 잘 구분하는 설명은 무엇입니까?

### Options / 보기
- **A.** They are always identical.
  - 한글: 둘은 항상 동일.
- **B.** Only training incurs cost; inference is free.
  - 한글: 학습에만 비용이 발생하고 추론은 무료.
- **C.** Only inference incurs cost; training is free.
  - 한글: 추론에만 비용이 발생하고 학습은 무료.
- **D.** Training is typically a large upfront/periodic cost (data prep + compute-intensive job); inference is an ongoing per-request cost (per-token or provisioned capacity) that scales with usage — both must be planned separately.
  - 한글: 학습은 대개 대규모 선행/주기적 비용(데이터 준비 + 컴퓨트 집약 작업); 추론은 사용량에 따라 확장되는 지속적 요청당 비용(토큰당 또는 프로비저닝된 용량) — 별도로 계획해야 함.

### ✅ Correct Answer / 정답
**D**

### 💡 Explanation / 해설

**English**:
Training = upfront capex-like; inference = ongoing opex-like. A, B, C misstate the model.

**한글**:
학습 = 선행 capex 성격; 추론 = 지속 opex 성격입니다. A·B·C는 잘못된 서술입니다.
