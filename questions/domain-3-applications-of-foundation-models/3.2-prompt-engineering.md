## Q207

### 🇺🇸 Question (English)
When picking examples for a few-shot prompt, which criterion is MOST important?

### 🇰🇷 문제 (한글)

- **A.** Pick the longest examples possible to fill the window.
  - 한글: 윈도우를 채우도록 최대한 긴 예시 선택.
- **B.** Pick random examples with no relation to the task.
  - 한글: 작업과 무관한 무작위 예시 선택.
- **C.** Pick a small, DIVERSE set of clean, correctly-formatted examples that COVER the shapes of inputs the model will actually see — including edge cases if relevant.
  - 한글: 정갈하고 형식이 맞으며 실제 볼 입력의 형상을 **다양하게 커버**하는 소수의 예시를 선택 — 필요 시 경계 사례 포함.

### 💡 Explanation / 해설

**한글**:

## Q209

### 🇺🇸 Question (English)
For tricky multi-step reasoning, one team asks the model to solve the same problem N times with temperature > 0 and takes the majority answer. Which technique is this?

### 🇰🇷 문제 (한글)

### Options / 보기
- **A.** Self-consistency — sampling several reasoning paths and choosing the most agreed-upon answer, which often improves accuracy on hard reasoning problems.

- **B.** Prompt injection.
  - 한글: 프롬프트 인젝션.
- **C.** Guardrails.
  - 한글: Guardrails.
- **D.** Batch transform.
  - 한글: 배치 트랜스폼.

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
Self-consistency samples multiple chains and votes on the answer. B is an attack. C is safety layer. D is a deployment mode.

**한글**:
자기 일관성은 여러 사슬을 샘플링해 답을 투표합니다. B는 공격, C는 안전 레이어, D는 배포 모드입니다.
