## Q001

### 🇺🇸 Question (English)
A junior engineer at a startup is trying to organize different AI technologies into a hierarchy for an internal training document. Which statement best describes the relationship between AI, Machine Learning (ML), Deep Learning (DL), and Generative AI (GenAI)?

### 🇰🇷 문제 (한글)
한 스타트업의 주니어 엔지니어가 사내 교육 자료를 위해 다양한 AI 기술을 계층 구조로 정리하려고 합니다. AI, 머신러닝(ML), 딥러닝(DL), 생성형 AI(GenAI) 사이의 관계를 가장 잘 설명하는 것은 무엇입니까?

### Options / 보기
- **A.** AI ⊃ ML ⊃ DL ⊃ GenAI (each is a subset of the previous)
  - 한글: AI ⊃ ML ⊃ DL ⊃ GenAI (뒤로 갈수록 앞의 하위 집합)
- **B.** ML and DL are the same thing; GenAI is a competing paradigm to AI
  - 한글: ML과 DL은 같은 개념이며, GenAI는 AI와 경쟁하는 별개의 패러다임
- **C.** GenAI ⊃ DL ⊃ ML ⊃ AI (GenAI is the broadest field)
  - 한글: GenAI ⊃ DL ⊃ ML ⊃ AI (GenAI가 가장 넓은 분야)
- **D.** AI, ML, DL, and GenAI are unrelated fields that developed independently
  - 한글: AI, ML, DL, GenAI는 각기 독립적으로 발전한 서로 무관한 분야

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
AI is the broadest field. ML is a subset of AI that learns from data. Deep Learning is a subset of ML that uses multi-layer neural networks. Generative AI is a subset of Deep Learning focused on creating new content, typically built on foundation models. B is wrong because ML and DL are distinct — DL uses neural networks while ML includes non-neural approaches. C reverses the hierarchy. D is factually incorrect.

**한글**:
AI가 가장 넓은 분야이며, ML은 데이터로부터 학습하는 AI의 하위 집합, 딥러닝은 다층 신경망을 사용하는 ML의 하위 집합, 생성형 AI는 파운데이션 모델 기반의 딥러닝 하위 집합입니다. B는 ML과 DL을 동일시하는 오류이고, C는 계층을 반대로, D는 사실과 다릅니다.

## Q002

### 🇺🇸 Question (English)
A team lead is explaining why a computer vision project for automated quality inspection uses Deep Learning rather than traditional Machine Learning. Which characteristic MOST justifies using Deep Learning for this image-based task?

### 🇰🇷 문제 (한글)
자동화된 품질 검사용 컴퓨터 비전 프로젝트에서 전통적 머신러닝 대신 딥러닝을 사용하는 이유를 팀 리드가 설명하고 있습니다. 이 이미지 기반 작업에 딥러닝 사용을 가장 잘 정당화하는 특성은 무엇입니까?

### Options / 보기
- **A.** Deep Learning always requires less computational resources than traditional ML.
  - 한글: 딥러닝은 항상 전통적 ML보다 적은 연산 자원을 요구합니다.
- **B.** Deep Learning automatically learns feature representations from raw pixels using multi-layer neural networks, removing the need for manual feature engineering.
  - 한글: 딥러닝은 다층 신경망을 이용해 원시 픽셀로부터 피처 표현을 자동으로 학습하여, 수동 피처 엔지니어링이 필요 없습니다.
- **C.** Deep Learning cannot process unstructured data, but is faster on structured data.
  - 한글: 딥러닝은 비정형 데이터를 처리할 수 없으나, 정형 데이터에서는 더 빠릅니다.
- **D.** Deep Learning replaces the need for labeled data completely.
  - 한글: 딥러닝은 라벨링된 데이터의 필요성을 완전히 없애 줍니다.

### ✅ Correct Answer / 정답
**B**

### 💡 Explanation / 해설

**English**:
Deep Learning's key advantage for images is automatic feature extraction through convolutional and other neural layers. Traditional ML would require humans to hand-craft features like edges, textures, or color histograms. A is wrong — DL usually requires MORE compute (GPUs). C is reversed — DL excels at unstructured data. D is wrong — supervised DL still needs labeled data.

**한글**:
이미지에 대한 딥러닝의 핵심 장점은 합성곱 등 신경망 레이어를 통한 자동 피처 추출입니다. 전통적 ML은 사람이 엣지·질감·색 히스토그램 같은 피처를 손수 설계해야 합니다. A는 반대로 DL이 GPU 등 더 많은 자원을 요구하므로 오답, C는 DL이 비정형 데이터에 강점이 있어 뒤집힌 설명, D는 지도 DL이 여전히 라벨을 필요로 하므로 오답입니다.

## Q003

### 🇺🇸 Question (English)
A product manager is drafting a document about "foundation models" for the executive team. Which description of a foundation model is MOST accurate?

### 🇰🇷 문제 (한글)
한 프로덕트 매니저가 경영진을 위해 "파운데이션 모델"에 관한 문서를 작성하고 있습니다. 파운데이션 모델에 대한 설명으로 가장 정확한 것은 무엇입니까?

### Options / 보기
- **A.** A rule-based expert system that requires no data for its knowledge base.
  - 한글: 지식 베이스를 위해 데이터가 필요 없는 규칙 기반 전문가 시스템.
- **B.** A small, task-specific model trained from scratch for a single narrow use case.
  - 한글: 하나의 좁은 유즈 케이스만을 위해 처음부터 학습된 소규모의 태스크 특화 모델.
- **C.** A large, pre-trained model that can be adapted to a wide range of downstream tasks through techniques like fine-tuning or prompting.
  - 한글: 대규모로 사전 학습되었으며, 파인튜닝이나 프롬프팅 같은 기법을 통해 다양한 하위 작업에 적응시킬 수 있는 모델.
- **D.** A hardware accelerator designed exclusively for training deep neural networks.
  - 한글: 딥 신경망 학습만을 위한 하드웨어 가속기.

### ✅ Correct Answer / 정답
**C**

### 💡 Explanation / 해설

**English**:
Foundation models are (1) trained on broad, large-scale data, (2) versatile — one model adapts to many tasks, and (3) adapted through fine-tuning/prompting/RAG rather than retraining from scratch. A describes symbolic AI (expert systems). B describes narrow single-task models — the opposite. D confuses a model with hardware like GPUs or AWS Inferentia.

**한글**:
파운데이션 모델의 정의는 (1) 광범위한 대규모 데이터로 학습, (2) 하나의 모델을 다양한 작업에 적응 가능, (3) 처음부터 재학습이 아닌 파인튜닝·프롬프팅·RAG로 적응입니다. A는 규칙 기반 전문가 시스템(심볼릭 AI)으로 무관, B는 반대인 좁은 특화 모델, D는 모델과 하드웨어(GPU/Inferentia)를 혼동한 설명입니다.

## Q004

### 🇺🇸 Question (English)
An analyst asks: "What makes a Large Language Model (LLM) specifically a 'language' model, as opposed to just any generative AI?" Which answer is MOST accurate?

### 🇰🇷 문제 (한글)
한 분석가가 다음과 같이 질문합니다: "대규모 언어 모델(LLM)이 그냥 생성형 AI가 아니라 특별히 '언어' 모델이 되는 이유는 무엇인가?" 가장 정확한 답변은 무엇입니까?

### Options / 보기
- **A.** LLMs only work on structured tabular data such as CSV files.
  - 한글: LLM은 CSV 같은 정형 표 형식 데이터에만 동작합니다.
- **B.** LLMs can only process one language at a time and never work across multiple languages.
  - 한글: LLM은 한 번에 하나의 언어만 처리할 수 있으며, 여러 언어에 걸쳐서는 동작하지 않습니다.
- **C.** LLMs are limited to translation tasks and cannot summarize or answer questions.
  - 한글: LLM은 번역 작업에만 국한되며, 요약이나 질의응답은 수행할 수 없습니다.
- **D.** LLMs are trained on massive text corpora to learn linguistic patterns, grammar, and context, so they specialize in understanding and generating human-like text.
  - 한글: LLM은 방대한 텍스트 말뭉치로 언어 패턴·문법·문맥을 학습하기 때문에, 인간과 유사한 텍스트를 이해·생성하는 데 특화됩니다.

### ✅ Correct Answer / 정답
**D**

### 💡 Explanation / 해설

**English**:
LLMs are a category of generative AI focused specifically on text. A is wrong: LLMs process unstructured text, not tabular data. B is wrong: modern LLMs are typically multilingual. C is wrong: LLMs perform many tasks (summarization, Q&A, code generation, classification). D captures the essential definition.

**한글**:
LLM은 텍스트에 특화된 생성형 AI의 한 종류입니다. A는 LLM이 정형 표 데이터가 아닌 비정형 텍스트를 처리하므로 오답, B는 최신 LLM이 대개 다국어이므로 오답, C는 LLM이 요약·질의응답·코드 생성 등 다양한 작업을 수행하므로 오답, D가 본질적 정의를 담고 있습니다.

## Q005

### 🇺🇸 Question (English)
A small retail company has a well-structured tabular dataset (about 5,000 rows of customer records with clean, labeled features like age, income, past purchase count) and wants to predict which customers are likely to churn. They have a limited budget and no GPU infrastructure. Which approach is MOST appropriate?

### 🇰🇷 문제 (한글)
소규모 소매 회사가 잘 정제된 정형 표 형식 데이터셋(나이, 소득, 과거 구매 횟수 등 라벨링된 피처가 있는 약 5,000행의 고객 레코드)을 보유하고 있으며, 어떤 고객이 이탈할지 예측하려고 합니다. 예산이 제한적이고 GPU 인프라도 없습니다. 가장 적합한 접근 방식은 무엇입니까?

### Options / 보기
- **A.** A large language model fine-tuned on the tabular data.
  - 한글: 표 형식 데이터로 파인튜닝한 대규모 언어 모델(LLM).
- **B.** A deep transformer model trained from scratch on the 5,000 rows.
  - 한글: 5,000행 데이터로 처음부터 학습시킨 딥 트랜스포머 모델.
- **C.** Traditional ML algorithms such as logistic regression or a decision tree/random forest.
  - 한글: 로지스틱 회귀나 의사결정 트리/랜덤 포레스트 같은 전통적 ML 알고리즘.
- **D.** A generative adversarial network (GAN) trained to hallucinate customer records.
  - 한글: 고객 레코드를 환각(hallucinate)해 내도록 학습한 생성적 적대 신경망(GAN).

### ✅ Correct Answer / 정답
**C**

### 💡 Explanation / 해설

**English**:
For small, clean, structured tabular data, traditional ML almost always beats deep learning in both accuracy and cost. Logistic regression, decision trees, and random forests train quickly on CPU and are interpretable. A is wrong: LLMs are for text, not tabular churn prediction. B is wrong: transformers need much larger datasets and expensive GPUs. D is wrong: GANs generate synthetic data — they do not predict a class like churn.

**한글**:
작고 정제된 정형 데이터에는 전통적 ML이 정확도·비용 모두에서 딥러닝을 거의 항상 이깁니다. A는 LLM이 텍스트용이라 부적합, B는 트랜스포머가 훨씬 큰 데이터와 GPU를 요구하므로 오답, D는 GAN이 합성 데이터를 생성할 뿐 이탈 같은 분류 예측을 수행하지 않습니다.

## Q006

### 🇺🇸 Question (English)
A company wants to build an email spam classifier. They have 100,000 emails, each already labeled as "spam" or "not spam" by human reviewers. Which type of machine learning is MOST appropriate for this task?

### 🇰🇷 문제 (한글)
한 회사가 이메일 스팸 분류기를 만들려고 합니다. 이미 사람이 각각 "스팸" 또는 "정상"으로 라벨링한 이메일 10만 개를 보유하고 있습니다. 이 작업에 가장 적합한 머신러닝 유형은 무엇입니까?

### Options / 보기
- **A.** Supervised learning
  - 한글: 지도학습(Supervised learning)
- **B.** Unsupervised learning
  - 한글: 비지도학습(Unsupervised learning)
- **C.** Reinforcement learning
  - 한글: 강화학습(Reinforcement learning)
- **D.** Self-directed learning without any input data
  - 한글: 입력 데이터 없이 자기 주도적으로 하는 학습

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
Supervised learning is correct because input data has explicit labels ("spam"/"not spam") and the goal is classification. B (unsupervised) is used for unlabeled data. C (reinforcement) is used when an agent takes actions and receives rewards. D is not a valid ML paradigm.

**한글**:
입력 데이터에 명시적 라벨("스팸"/"정상")이 있고 분류 작업이므로 지도학습이 정답입니다. B(비지도학습)는 라벨 없는 데이터에서 패턴을 찾을 때 사용하며, C(강화학습)는 에이전트가 행동·보상을 받는 방식으로 부적합, D는 유효한 ML 패러다임이 아닙니다.

## Q007

### 🇺🇸 Question (English)
A marketing team has a dataset of 500,000 customer transactions with no pre-defined groups. They want to discover natural customer segments to design targeted campaigns. Which type of learning is MOST appropriate?

### 🇰🇷 문제 (한글)
한 마케팅 팀이 사전 정의된 그룹이 없는 50만 건의 고객 거래 데이터셋을 가지고 있습니다. 타깃 캠페인 설계를 위해 자연스러운 고객 세그먼트를 발견하고자 합니다. 가장 적합한 학습 유형은 무엇입니까?

### Options / 보기
- **A.** Supervised learning with all customers labeled as "important"
  - 한글: 모든 고객을 "중요"로 라벨링한 지도학습
- **B.** Unsupervised learning (e.g., clustering)
  - 한글: 비지도학습 (예: 클러스터링)
- **C.** Reinforcement learning with random rewards
  - 한글: 무작위 보상을 사용하는 강화학습
- **D.** No learning is needed — just sort customers alphabetically
  - 한글: 학습이 필요 없이 고객을 알파벳순으로 정렬

### ✅ Correct Answer / 정답
**B**

### 💡 Explanation / 해설

**English**:
No pre-defined labels and the goal is to discover hidden groupings — this is textbook unsupervised learning, with clustering (e.g., k-means) as the canonical method. A is nonsensical: labeling everyone identically provides no signal. C mischaracterizes RL — it needs a meaningful reward signal. D isn't machine learning at all.

**한글**:
사전 정의된 라벨이 없고 숨겨진 그룹을 발견하는 것이 목표이므로 비지도학습이며, k-평균 같은 클러스터링이 표준입니다. A는 모두 같은 라벨을 붙이면 지도학습이 성립하지 않으므로 오답, C는 강화학습이 의미 있는 보상 신호를 요구하므로 부적합, D는 머신러닝조차 아닙니다.

## Q008

### 🇺🇸 Question (English)
An engineer is experimenting with AWS DeepRacer, a service where a virtual car learns to drive around a track by taking actions and receiving rewards for staying on the road. Which type of machine learning does DeepRacer primarily demonstrate?

### 🇰🇷 문제 (한글)
한 엔지니어가 AWS DeepRacer를 실험하고 있습니다. 가상 자동차가 트랙을 따라 주행하는 법을 배우기 위해 행동을 취하고, 도로 위에 머무르면 보상을 받는 서비스입니다. DeepRacer가 주로 보여주는 머신러닝 유형은 무엇입니까?

### Options / 보기
- **A.** Supervised learning with labeled steering data
  - 한글: 조향 데이터가 라벨링된 지도학습
- **B.** Unsupervised clustering of track sensor values
  - 한글: 트랙 센서 값의 비지도 클러스터링
- **C.** Reinforcement learning
  - 한글: 강화학습(Reinforcement learning)

## Q030

### 🇺🇸 Question (English)
How does a Generative Adversarial Network (GAN) work at a high level?

### 🇰🇷 문제 (한글)
생성적 적대 신경망(GAN)은 고수준에서 어떻게 동작합니까?

### Options / 보기
- **A.** A single neural network memorizes every training example and reproduces them at inference time.
  - 한글: 하나의 신경망이 모든 학습 예제를 암기하고 추론 시 그대로 재생성.
- **B.** Two identical copies of the same model are run in parallel and averaged.
  - 한글: 동일한 모델의 두 사본을 병렬로 실행해 평균을 취함.
- **C.** A generator network creates synthetic samples and a discriminator network tries to tell them apart from real samples; the two are trained in competition, and the generator improves as the discriminator gets harder to fool.
  - 한글: 생성기 네트워크가 합성 샘플을 만들고, 판별기 네트워크가 그것을 실제 샘플과 구분하려 합니다. 이 둘을 경쟁시켜 학습하며, 판별기가 속기 어려워질수록 생성기 품질이 향상됩니다.
- **D.** A single generator network samples random numbers with no training required.
  - 한글: 하나의 생성기 네트워크가 학습 없이 무작위 수를 샘플링.

### ✅ Correct Answer / 정답
**C**

### 💡 Explanation / 해설

**English**:
GANs pair a generator (creates synthetic data) with a discriminator (classifies real vs. synthetic). Training is adversarial — the generator learns to fool the discriminator, and the discriminator learns to catch fakes. As they compete, sample quality improves. A confuses generation with memorization. B describes ensembling. D omits the training and the discriminator entirely.

**한글**:
GAN은 생성기(합성 데이터 생성)와 판별기(실제 vs 합성 분류)를 쌍으로 학습합니다. 적대적 학습으로, 생성기는 판별기를 속이도록 배우고 판별기는 가짜를 잡아내도록 학습합니다. 경쟁하면서 샘플 품질이 향상됩니다. A는 생성과 암기 혼동, B는 앙상블 설명, D는 학습·판별기가 통째로 빠진 오답입니다.
