## Q060

### 🇺🇸 Question (English)
A team's model achieves 99% accuracy on training data but only 60% on validation data. Which problem is this MOST likely?

### 🇰🇷 문제 (한글)
한 팀의 모델이 학습 데이터에서 99% 정확도를 달성했지만 검증 데이터에서는 60%밖에 나오지 않습니다. 가장 가능성이 높은 문제는 무엇입니까?

### Options / 보기
- **A.** Overfitting — the model memorized training data instead of learning general patterns.
  - 한글: 오버피팅(과적합) — 모델이 일반 패턴을 배우는 대신 학습 데이터를 암기함.
- **B.** Underfitting — the model is too simple to capture the pattern.
  - 한글: 언더피팅(과소적합) — 모델이 패턴을 담기에 너무 단순함.
- **C.** Concept drift caused by production data.
  - 한글: 프로덕션 데이터에 의한 개념 드리프트.
- **D.** A bug in the SageMaker endpoint routing.
  - 한글: SageMaker 엔드포인트 라우팅 버그.

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
A large gap between high training accuracy and low validation accuracy is the textbook signature of overfitting. B would produce low accuracy on BOTH sets. C occurs post-deployment, not during validation. D would break inference, not skew accuracy this way.

**한글**:
학습 정확도는 높지만 검증 정확도는 크게 낮은 격차는 오버피팅의 교과서적 신호입니다. B는 학습·검증 모두 낮게 나오고, C는 배포 후 발생하며, D는 추론을 망가뜨릴 뿐 이런 방식의 편차를 만들지 않습니다.

### 🇺🇸 Question (English)
A model achieves only 55% accuracy on BOTH training and validation data — well below acceptable. Which of the following is MOST likely to help?

### 🇰🇷 문제 (한글)
한 모델이 학습·검증 데이터 모두에서 55%의 낮은 정확도만 달성합니다. 이 문제 해결에 가장 도움이 될 가능성이 높은 것은 무엇입니까?

### Options / 보기
- **A.** Reduce the training dataset size.
  - 한글: 학습 데이터셋 크기 축소.
- **B.** Increase model capacity (deeper/wider network) or train longer with more data and better features.
  - 한글: 모델 용량 증가(더 깊거나 넓은 네트워크) 또는 더 많은 데이터·좋은 피처로 더 오래 학습.

  - 한글: 모든 검증을 비활성화하고 학습 정확도가 100%가 될 때까지 재학습.

**한글**:

## Q062

### 🇺🇸 Question (English)
When calling an LLM on Amazon Bedrock, which effect does INCREASING the `temperature` parameter have on outputs?

### 🇰🇷 문제 (한글)
Amazon Bedrock에서 LLM을 호출할 때, `temperature` 파라미터를 **높이면** 출력에 어떤 효과가 있습니까?

### Options / 보기
- **A.** Outputs become more deterministic and repetitive.
  - 한글: 출력이 더 결정론적이고 반복적으로 변함.
- **B.** Outputs are guaranteed to be factually accurate.
  - 한글: 출력이 사실적 정확성을 보장받음.
- **C.** Outputs become more diverse and creative, at the cost of predictability.

## Q063

### 🇺🇸 Question (English)
Which BEST describes how the `Top-P` (nucleus sampling) parameter works when generating text?

### 🇰🇷 문제 (한글)
텍스트 생성 시 `Top-P`(뉴클리어스 샘플링) 파라미터가 동작하는 방식을 가장 잘 설명한 것은 무엇입니까?

### Options / 보기
- **A.** It caps the number of GPU cores used.
  - 한글: 사용하는 GPU 코어 수를 제한.
- **B.** It fixes the exact number of candidate tokens to a value P at every step.
  - 한글: 매 단계마다 후보 토큰의 개수를 P로 고정.
- **C.** It converts probabilities to logits.
  - 한글: 확률을 로짓으로 변환.
- **D.** It selects from the smallest set of tokens whose cumulative probability meets P (an adaptive number of tokens per step).
  - 한글: 누적 확률이 P가 되는 가장 작은 토큰 집합에서 선택 (매 단계마다 적응적으로 토큰 수가 달라짐).

### ✅ Correct Answer / 정답
**D**

### 💡 Explanation / 해설

**English**:
Top-P uses cumulative probability — the set of tokens is adaptive per step, containing enough tokens to reach P. This differs from Top-K, which fixes the count. A and C invent behavior. B describes Top-K, not Top-P.

**한글**:
Top-P는 누적 확률을 사용하며, 매 단계마다 P에 도달하는 만큼의 적응적 토큰 집합을 만듭니다. 이는 개수를 고정하는 Top-K와 다릅니다. A·C는 지어낸 설명, B는 Top-P가 아닌 Top-K 설명입니다.

### 🇺🇸 Question (English)
Which metric is the harmonic mean of precision and recall, useful when you want a single number balancing both?

### 🇰🇷 문제 (한글)
정밀도와 재현율의 조화 평균이며, 둘 사이의 균형을 하나의 숫자로 표현하고 싶을 때 유용한 지표는 무엇입니까?

### Options / 보기
- **A.** RMSE
  - 한글: RMSE
- **B.** Perplexity
  - 한글: Perplexity

  - 한글: F1 Score
- **D.** ROC AUC
  - 한글: ROC AUC

### ✅ Correct Answer / 정답
**C**

### 💡 Explanation / 해설

**English**:
F1 = 2·(P·R)/(P+R) — harmonic mean balancing precision and recall. A is regression error. B is language modeling. D measures ranking quality across thresholds, not the P/R harmonic mean.

**한글**:
F1 = 2·(P·R)/(P+R) — 정밀도와 재현율의 조화 평균입니다. A는 회귀 오차, B는 언어 모델링, D는 임계값에 걸친 랭킹 품질 측정으로 조화 평균이 아닙니다.

## Q080

### 🇺🇸 Question (English)
Which AWS service is designed to orchestrate end-to-end ML workflows (data prep → training → evaluation → registration → deployment) as a repeatable pipeline?

### 🇰🇷 문제 (한글)
데이터 준비 → 학습 → 평가 → 등록 → 배포까지 엔드투엔드 ML 워크플로를 반복 가능한 파이프라인으로 오케스트레이션하도록 설계된 AWS 서비스는 무엇입니까?

### Options / 보기
- **A.** SageMaker Pipelines
  - 한글: SageMaker Pipelines
- **B.** Amazon Kendra
  - 한글: Amazon Kendra
- **C.** AWS Config
  - 한글: AWS Config
- **D.** Amazon Polly
  - 한글: Amazon Polly

### ✅ Correct Answer / 정답
**A**

### 💡 Explanation / 해설

**English**:
SageMaker Pipelines orchestrates repeatable end-to-end ML workflows and integrates with Model Registry for MLOps. B is search, C tracks AWS resource configuration, D is TTS — none orchestrate ML pipelines.

**한글**:
SageMaker Pipelines는 반복 가능한 엔드투엔드 ML 워크플로를 오케스트레이션하며 Model Registry와 통합됩니다. B는 검색, C는 AWS 리소스 구성 추적, D는 TTS로 ML 파이프라인 오케스트레이션과 무관합니다.
